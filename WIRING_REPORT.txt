ARCHITECTURE DECISION
---------------------
The repose package (Intel_feed/Event_monitor/Observer) lives INSIDE the repose-runtime
container (/opt/repose/repose, bind-mounted from
/opt/repose, uid 10010), where its
dependencies, Bitwarden SDK config, and Redis network reachability exist.
The host has no repose venv. Therefore each new host systemd unit follows
the existing [operator]-*.service pattern (Type=simple, User=[operator], Restart=always,
RestartSec=10, journal) but its ExecStart runs `docker exec -u 10010` into
the container, executing the hermes venv python with PYTHONPATH=/opt/repose.
[operator] is in the docker group, so no privilege change was needed.
No secrets are passed via environment — all secrets resolve via the
Bitwarden SDK inside the container (constraint honored).

REPOSE PACKAGE INSTALL (find_spec('repose') == None)
----------------------------------------------------
Root cause IDENTIFIED: the editable install's import-hook finder had an
empty MAPPING ({}), because pyproject.toml used
  [tool.setuptools.packages.find] include = ["repose*"]
against a FLAT layout where the project root dir itself is the `repose`
package (children are agents/, cli/, utils/ — none match "repose*"), so
zero packages were discovered.

Attempted the proper fix (pyproject explicit package list + package-dir,
then `pip install -e --no-deps --no-build-isolation`). It FAILED with
`invalid command 'bdist_wheel'` — the `wheel` package is absent from the
hermes-agent venv, which is SHARED infrastructure. Per the task constraint
("if the install fails or causes any error in the existing package state,
revert and document — do not force it"), pyproject.toml was REVERTED to its
pre-wire snapshot and the shared venv was left untouched. The failed pip
attempt ran in a temp build dir and did NOT alter the existing install
(pip show repose still reports v1.0.0 editable; artifacts unchanged).

OUTCOME: install fix SKIPPED (documented). Runtime is fully solved without
it via per-process PYTHONPATH=/opt/repose (scoped to the three new
systemd units only — zero blast radius on the shared venv / hermes-agent).
`repose` imports and the CLI run correctly under that PYTHONPATH.
